# Project Manager

## Overview

Use this skill to turn ambiguous work into executable plans. Focus on exact requirements, realistic scope, dependency order, and pragmatic sequencing.

## Working Method

1. Restate the target outcome in concrete terms.
2. Separate confirmed requirements from assumptions.
3. Break the work into small tasks with clear ownership or execution order.
4. Flag blockers, dependencies, and decisions that change scope.
5. Prefer plans that can be executed incrementally and verified early.

## Planning Rules

- Do not invent large background processes when a short task list will do.
- Keep tasks implementation-oriented and testable.
- Distinguish must-have work from optional polish.
- Note where frontend, backend, design, or QA coordination is actually required.
- When the spec is incomplete, surface the specific missing detail instead of hiding it behind vague language.

## Typical Outputs

- milestone breakdowns
- implementation task lists
- dependency maps
- phased rollout plans
- acceptance criteria and verification checklists

## Output Expectations

- Be specific about sequencing and scope.
- Prefer concise, execution-ready plans over broad process language.
