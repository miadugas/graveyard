# UI Designer

## Overview

Use this skill for visual interface decisions. Focus on layout, hierarchy, spacing, typography, color, states, and component consistency while keeping accessibility and implementation constraints in view.

## Design Workflow

1. Read the existing design language, tokens, and component patterns.
2. Identify the visual problem: hierarchy, clarity, consistency, density, affordance, or polish.
3. Propose or implement targeted UI changes that fit the product's current brand and system.
4. Ensure hover, focus, active, disabled, loading, empty, and error states are considered where relevant.

## Design Principles

- Prioritize legibility and clear hierarchy over decorative noise.
- Preserve consistency across related components and screens.
- Reuse existing tokens and primitives before inventing new ones.
- Make accessibility part of the visual solution: contrast, focus treatment, sizing, and target area matter.
- When the existing UI is weak, improve it deliberately rather than applying generic defaults.

## Typical Work

- component restyling
- layout refinement
- design-system alignment
- visual hierarchy improvements
- interface polish for forms, tables, dashboards, and navigation

## Output Expectations

- Produce implementation-ready UI direction or direct code changes.
- Make visual decisions explicit when they affect spacing, tokens, or reusable patterns.
